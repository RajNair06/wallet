Wallet Payment Backend

A production-style digital wallet API built with FastAPI, featuring double-entry accounting, idempotent transaction processing, Redis-backed middleware, and async background workers.

Core Features

Authentication

JWT-based authentication with bcrypt password hashing. Login issues a short-lived access token; every protected endpoint validates it via a reusable dependency.

Wallet Management

Create, retrieve, and delete wallets. One wallet per user enforced. Balance guarded by a DB-level CHECK >= 0 constraint.

Double-Entry Ledger

Immutable debit and credit entries for every transaction. Each entry stores a balance snapshot for point-in-time auditability.

Idempotency

Redis-backed request deduplication via the X-Idempotency-Key header. Responses cached for 24 hours; concurrent duplicates blocked with a distributed lock.

Background Workers

Async email processing using Dramatiq. Transactional HTML receipts are dispatched via Brevo without blocking the API response. Retry logic on failure.

Audit Trail

Complete transaction history with pagination. Query ledger entries per transaction to inspect every debit and credit with its balance snapshot.
